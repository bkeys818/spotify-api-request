getUserTopArtistsandTracks

  • getUserTopArtistsandTracks<T>(token: Token | string, type: T, options?: { limit?: number; offset?: number; timeRange?: "long_term" | "medium_term" | "short_term" }): Promise<PagingObject<T extends "artists" ? ArtistObject : TrackObject, `my top ${T}`>>

  • Get the current user’s top artists or tracks based on calculated affinity.

    Type parameters

    • T: "artists" | "tracks"

    Parameters

    • token: Token | string

      A valid access token from the Spotify Accounts service: see the Web API Authorization Guide for details. The access token must have been issued on behalf of the current user.
      Getting details of a user’s top artists and tracks requires authorization of the user-top-read scope. See Using Scopes.

    • type: T

      The type of entity to return.

    • Optional options: { limit?: number; offset?: number; timeRange?: "long_term" | "medium_term" | "short_term" }
      • Optional limit?: number

        The number of entities to return. Default: 20. Minimum: 1. Maximum: 50.

      • Optional offset?: number

        The index of the first entity to return. Default: 0 (i.e., the first track). Use with limit to get the next set of entities.

      • Optional timeRange?: "long_term" | "medium_term" | "short_term"

        Over what time frame the affinities are computed.

        Valid values:

        • long_term - Calculated from several years of data and including all new data as it becomes available.
        • medium_term - Approximately last 6 months. (Default)
        • short_term - Approximately last 4 weeks.

    Returns Promise<PagingObject<T extends "artists" ? ArtistObject : TrackObject, `my top ${T}`>>

    `>>} A {@link PagingObject paging object} of {@link ArtistObject Artists} or {@link TrackObject Tracks}.
