getMultipleShows

  • getMultipleShows(token: Token | string, ids: string[], options?: { market: string }): Promise<{ shows: SimplifiedShowObject[] }>

  • Get Spotify catalog information for several shows based on their Spotify IDs.

    Parameters

    • token: Token | string

      A valid access token from the Spotify Accounts service: see the Web API Authorization Guide for details.

    • ids: string[]

      A list of the Spotify IDs for the shows. Maximum: 50 IDs.

    • Optional options: { market: string }
      • market: string

        An ISO 3166-1 alpha-2 country code. If a country code is specified, only shows and episodes that are available in that market will be returned.

        If a valid user access token is specified in the request header, the country associated with the user account will take priority over this parameter.

        Note: If neither market or user country are provided, the content is considered unavailable for the client.

        Users can view the country that is associated with their account in the account settings.

    Returns Promise<{ shows: SimplifiedShowObject[] }>

    } An object whose key is shows and whose value is an array of {@link SimplifiedShowObject simple show object}.

getShowEpisodes

  • getShowEpisodes(token: Token | string, id: string, options?: { limit?: number; market?: string; offset?: number }): Promise<PagingObject<SimplifiedEpisodeObject, "show’s episodes">>

  • Get Spotify catalog information about an show’s episodes. Optional parameters can be used to limit the number of episodes returned.

    Parameters

    • token: Token | string

      valid access token from the Spotify Accounts service: see the Web API Authorization Guide for details.

    • id: string

      The Spotify ID for the show.

    • Optional options: { limit?: number; market?: string; offset?: number }
      • Optional limit?: number

        The maximum number of episodes to return. Default: 20. Minimum: 1. Maximum: 50.

      • Optional market?: string

        An ISO 3166-1 alpha-2 country code. If a country code is specified, only shows and episodes that are available in that market will be returned.

        If a valid user access token is specified in the request header, the country associated with the user account will take priority over this parameter.

        Note: If neither market or user country are provided, the content is considered unavailable for the client.

        Users can view the country that is associated with their account in the account settings.

      • Optional offset?: number

        The index of the first episode to return. Default: 0 (the first object). Use with limit to get the next set of episodes.

    Returns Promise<PagingObject<SimplifiedEpisodeObject, "show’s episodes">>
